git使用技巧

Posted on 2024-10-08

1 git使用ssh密钥进行github认证

参考官方文档：https://docs.github.com/zh/authentication/connecting-to-github-with-ssh/about-ssh

1 是否存在现有的ssh密钥

输入ls -al ~/.ssh以查看是否存在现有的 SSH 密钥。

2 生成ssh密钥对

粘贴以下文本，将示例中使用的电子邮件替换为 GitHub 电子邮件地址。
ssh-keygen -t ed25519 -C "youremail@163.com"

当系统提示您“Enter a file in which to save the key（输入要保存密钥的文件）”时，可以按 Enter 键接受默认文件位置。 请注意，如果以前创建了 SSH 密钥，则 ssh-keygen 可能会要求重写另一个密钥，在这种情况下，我们建议创建自定义命名的 SSH 密钥。 为此，请键入默认文件位置，并将 id_ALGORITHM 替换为自定义密钥名称。

3 将 SSH 密钥添加到 ssh-agent

  1. 在后台启动 ssh 代理
    eval "$(ssh-agent -s)"
  2. 如果你使用的是 macOS Sierra 10.12.2 或更高版本，则需要修改 ~/.ssh/config 文件以自动将密钥加载到 ssh-agent 中并在密钥链中存储密码。
    open ~/.ssh/config

打开你的 ~/.ssh/config 文件，添加
Host github.com AddKeysToAgent yes IdentityFile ~/.ssh/id_ed25519

ssh-add ~/.ssh/id_ed25519

4 新增 SSH 密钥到 GitHub 帐户

  1. 将 SSH 公钥复制到剪贴板
    $ pbcopy < ~/.ssh/id_ed25519.pub
  2. 点击头像，点击设置，点击ssh密钥，添加密钥，粘贴，完成

5 测试连接

确保已经完成：

  1. 检查现有 SSH 密钥

  2. 本地生成新 SSH 密钥

  3. 为 GitHub 帐户添加新的 SSH 密钥

  4. 打开终端ssh -T git@github.com
    Hi hechongyang1101! You’ve successfully authenticated, but GitHub does not provide shell access.

  5. 验证生成的消息包含您的用户名

github遇到的问题

1 Error: Spawn failed

HTTP 400 curl 22 The requested URL returned error: 400 send-pack: unexpected disconnect while reading sideband packet
When pushing a large amount of data (initial push of a big repository, change with very big file(s)) may require a higher http.postBuffer setting on your git client (not the server). Increase the Git buffer size to the largest individual file size of your repo:

git config --global http.postBuffer 524288000
